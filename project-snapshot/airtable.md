## Airtable Access Pattern

- Reads generally go through `AirtableService.getRecords(tableKey)`.
- Writes generally go through `AirtableAdapter`, which translates internal fields through `AIRTABLE_TABLE_SCHEMAS`.
- `AirtableClient` implements pagination, throttling, create/update/upsert, and Airtable error mapping.

## Current Table Role Summary

- `Spotify_Episode_Metrics`: raw Spotify CSV/source table used by Spotify Channel Breakdown.
