## Purpose

This repository implements a Communications Intelligence Platform: a KPI dashboard that aggregates communications analytics from external systems into Airtable, normalizes them in a Node/TypeScript backend, and exposes dashboard-ready REST APIs to a static web dashboard.

## Current Architecture

```
External APIs / CSV imports
  -> Airtable tables
  -> backend services and calculation models
  -> REST APIs
  -> static dashboard in apps/backend/public
```

The backend owns business logic. The browser UI calls REST endpoints and should not need platform-specific source knowledge.

## Technologies

- Runtime: Node.js >=20.11.0
- Backend: TypeScript, Express, built-in `fetch`, Airtable REST API
- Frontend in current repo: static HTML/CSS/JavaScript served by Express from `apps/backend/public`
- Deployment target: Render, via `render.yaml`
- Data store: Airtable

## Current Completion Status

- Backend and static dashboard are implemented.
- Airtable service, adapter, schema mapping, caching, and diagnostics are implemented.
- KPI scoring and content-ranking models are implemented in backend services.
- Mailchimp has a live API service that writes email metrics to `KPI_History`.
- Spotify CSV ingestion and Spotify dashboard reads are in transition: the dashboard reads `Spotify_Episode_Metrics`, while the current ingestion connector code still writes to `spotifyWeeklySnapshot`.
- Castos, YouTube, Website, Facebook, and Instagram connectors are placeholders or future registrations.
- `apps/dashboard` exists but is empty; there is no React app in the current repository state.

## Development Philosophy Observed

- Keep raw/source data in Airtable.
- Normalize and score in backend services.
- Keep frontend API consumers platform-agnostic.
- Preserve legacy routes/tables while new communications tables are connected.
- Prefer schema mappings at the Airtable boundary.
